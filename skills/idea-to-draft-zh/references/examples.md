# 用例示例

## 示例 1：主题到初稿

### 输入

- 主题：为什么个人开发者应该先做 OpenClaw 技能，而不是一上来做大平台
- 受众：想做 AI 副业的技术人
- 约束：
  - 中文
  - 口语化但不空泛
  - 800 到 1200 字

### 期望形态

1. 受众
2. 核心承诺
3. 角度备选
4. 最终结构
5. 初稿
6. 下一轮修改建议

### 示例输出片段

## 2. 核心承诺

- 让读者理解，先做技能更容易拿到真实反馈、真实用户和真实案例。

## 3. 角度备选

- 角度 A：从成本视角切入
- 角度 B：从政策审核友好度切入
- 角度 C：从个人执行难度切入

## 示例 2：笔记到初稿

### 输入

- 原始笔记：
  - OpenClaw 还在早期
  - 技能比平台轻
  - 更容易做演示
  - 政策第一条直接支持技能包

### 输出备注

- 先收束一个中心论点，再开始写。
- 不要把多个角度混成一篇散文。
