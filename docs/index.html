先做窄 skill，再做平台。先交付证据，再讲故事。

这个仓库把三个聚焦的 OpenClaw 中文 skills 组织成一套可持续扩展的起步结构， 先覆盖研究资料整理、内容起草和周报输出，再逐步扩展成更完整的工作流体系。

当前结构

  • 3 个可继续打磨的 starter skills
  • 中文优先的 prompt 与模板
  • 可直接演示的样例输出
  • 适合继续接 GitHub Pages 与 ClawHub

定位

这是什么

这是一个面向中文场景的 OpenClaw skills 仓库，不是“大而全的平台”。 它更像一个持续积累的作品库：每个 skill 先解决一个明确问题，方便演示、测试和迭代。

为什么这样做

为什么先走这条路

在早期生态里，真正有价值的往往不是抽象的大平台叙事，而是可运行、可演示、可复用的组件。 窄 skill 更快做出来，也更容易被别人理解。

起步 Skills

当前这三项

它们先组成一条最小闭环：资料整理、结构收束、周期性输出。

01 / research-digest-zh

研究摘要

把分散资料整理成一份带来源意识的中文研究摘要，输出关键发现、含义判断和待确认问题。

  • 适合政策解读、主题调研、竞品扫描
  • 把证据和判断分开
  • 输出面向决策的摘要
资料 -> 发现 -> 判断

02 / idea-to-draft-zh

想法到初稿

把一个主题或一堆原始笔记收束成中文初稿，强制经过受众、角度、结构和修改建议这几步。

  • 适合文章、笔记、观点输出起稿
  • 先做角度选择，再开始写
  • 目标是可用初稿，不是空泛 brainstorm
主题 -> 角度 -> 提纲 -> 初稿

03 / source-to-weekly-brief-zh

周简报

把一周内的信息更新整理成一份紧凑的中文简报，重点突出发生了什么变化、为什么重要、下一步做什么。

  • 适合做生态跟踪、周报、信息巡检
  • 优先写变化，不重复背景
  • 最后收束成具体动作
周内输入 -> 变化摘要 -> 下一步动作

为什么它不是空壳

每个 skill 都已经有最小可发布结构

Skill 说明

`SKILL.md` 定义触发条件、工作流、规则和输出结构。

真实用例

`references/examples.md` 放了测试用例和预期形态，方便后续验证。

模板

`templates/` 保持输出结构稳定，方便比较、打分和迭代。

演示样例

`assets/sample-output-*.md` 让这个仓库一开始就能拿来展示。

下一步

接下来怎么继续长

1

做真实测试

先做显式调用，再测试 OpenClaw 能不能在隐式场景下选对 skill。

2

收紧输出质量

找出弱段落、减少泛化表达，并给每个 skill 再补一条更强的真实示例。

3

选择性发布

持续维护这个 GitHub 仓库，并筛选哪些 skill 适合继续发布到 ClawHub。
